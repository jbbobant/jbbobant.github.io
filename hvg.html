Cell Sentences & HVG Evaluation

Applying Language Models to Single-Cell Genomics

View GitHub Repo Read Research Report

The Research Objective

HVG vs. Full Genome

In single-cell analysis, Highly Variable Gene (HVG) selection is a standard step to reduce noise by focusing on the ~2000 genes with the highest variance.

This project's core question was to evaluate the trade-off of this data reduction. We compared the performance of models trained on "cell sentences" built from only the top 2000 HVGs against models trained on sentences built from the full genome.

The hypothesis: The HVG-only sentences would perform just as well on classification tasks, proving that the most critical biological signal is contained within this subset.

HVG and full genome cell sentence construction process.

Downstream Tasks

Task A: Gene Reconstruction

  • Model: Linear Regression
  • Objective: Prove "reversibility" of embedding
  • Loss: Mean Squared Error (MSE)
  • Metrics: R2, Pearson, Spearman

Task B: Cell-Type Annotation

  • Model: 2-Layer MLP Classifier
  • Objective: Classify cell identity
  • Loss: Cross-Entropy
  • Metrics: Accuracy, Macro-F1

Key Results

Task 1: Gene Expression Reconstruction

This task tested the "reversibility" of the cell sentences. A simple linear regression model was trained to predict the original gene expression profile using only the cell embeddings.

Example scatter plot for Skin showing strong correlation between true and reconstructed expression.

Result: The model achieved an average R2 of 0.85 and Pearson Correlation of 0.92, confirming that the rank-based sentence embedding retains a high degree of quantitative expression information.

Task 2: Cell-Type Annotation

This task trained a simple MLP classifier on the cell embeddings to predict the cell type (e.g., fibroblast, T cell). This was the primary test for the HVG vs. Full Genome comparison.

Macro-F1 score for inter-dataset cell-type annotation.

Result: The models achieved state-of-the-art performance (e.g., Macro-F1 of ~0.89). Most importantly, the HVG-only model performed almost identically to the full-genome model, validating our hypothesis.

Key Takeaways & Learnings

This project was a deep dive into the intersection of MLOps and computational biology. Beyond the research findings, the key engineering takeaway was refactoring a complex, multi-notebook research project into a clean, testable, and reproducible codebase.

  • ML Engineering: Successfully modularized the entire pipeline, separating data preprocessing, model definition, and training loops into distinct, reusable classes.
  • Bioinformatics Pipeline: Gained hands-on experience with `scanpy` and `anndata` for handling scRNA-seq data, including QC filtering, normalization, and HVG selection.
  • Reproducible Science: Implemented a config-driven workflow (`yaml`) and global seeding (`seed_everything`) to ensure all results are 100% reproducible.
  • Advanced PyTorch: Built custom `Trainer` classes for both classification and regression, implementing early stopping and appropriate metric tracking (Macro-F1, R2) for each task.
