Wine Vintage Quality Forecasting

Predicting Wine Quality from Weather and Grape Data using Deep Learning

The Research Objective

Forecasting Quality Without Tasting

Even for wine enthusiasts, determining the quality of a vintage without tasting is a significant challenge. This project was motivated by a desire to leverage data to solve this problem.

The objective was to combine wine-specific data (grape variety, region, appellation) with geospatial and temporal data (meteorological conditions, soil composition) to predict its consumer-based quality rating.

This involved a complex data engineering pipeline to build a unique tabular dataset, followed by a comparative analysis of classical, MLP, and Transformer-based models.

Data Sources

The project's foundation is a novel dataset created by fusing three distinct data sources through a custom ETL pipeline.

Vivino Wine Data

Scraped data for thousands of wines, including grape, region, vintage, and user ratings.

Meteorological Data

Historical weather data (e.g., precipitation, temperature) sourced from APIs for each wine region.

Geospatial & Soil Data

Geocoded AOC (Appellation) boundaries merged with SoilGrids data for soil composition.

Key Results

Baseline Performance

Classical models like XGBoost and Random Forest served as strong baselines. XGBoost, in particular, achieved a robust Macro-F1 score of 0.81, setting a high bar for the deep learning models.

Comparison of basline models performance metrics.

MLP Outperforms Transformers

While SOTA models like FT-Transformer and TabNet were tested, the best-performing model was a well-tuned Multi-Layer Perceptron (MLP).

Results for the MLP model.

Result: The MLP achieved the highest Macro-F1 score, demonstrating that for this specific tabular dataset, a simpler deep learning architecture (optimized with Optuna) was more effective than more complex Transformer-based models.

Key Takeaways & Learnings

This project was an end-to-end exercise in applied data science and MLOps. The primary challenge was not just model training, but the complex data engineering required to build the dataset from scratch.

  • MLOps Engineering: Refactored a complex data pipeline from notebooks into a modular, config-driven project (`pyproject.toml`, `src/` layout).
  • Heterogeneous Data ETL: Built a full data pipeline, including web scraping (`Vivino`), API integration (`Meteo`), and geospatial data merging (AOC coordinates & `SoilGrids`).
  • Tabular Deep Learning: Compared SOTA tabular models (MLP, TabNet, FT-Transformer) against classical methods (XGBoost), finding MLP was most effective.
  • Hyperparameter Tuning: Implemented `Optuna` for automated hyperparameter search to optimize the MLP architecture and training parameters.
